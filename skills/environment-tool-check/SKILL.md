# 环境与工具检查

## 适用场景

当用户遇到以下问题时，优先使用本技能：

- 命令无法执行
- 程序找不到依赖或可执行文件
- 服务、接口或端口连不上
- 不确定当前机器环境是否满足要求
- 需要先做快速健康检查再继续执行任务

## 推荐工具

- `terminal-tools_get-system-info`
- `terminal-tools_which-command`
- `terminal-tools_get-env-variable`
- `network-tools_ping`
- `network-tools_telnet`
- `network-tools_get-local-ip`
- `network-tools_get-public-ip`

如果当前 Profile 没有启用这些环境诊断工具，应先说明当前无法直接完成环境检查，并提示用户启用对应工具或切换到具备这些工具的 Profile。

## 工作流

1. 先明确问题属于哪一类：命令问题、环境变量问题、网络问题还是服务连通性问题。
2. 如果是命令问题，先确认命令是否存在，再看 PATH 或相关环境变量。
3. 如果是网络问题，先检查基本连通性，再看端口是否可达。
4. 如果是环境问题，先收集系统信息和关键变量，再判断是否缺少依赖或配置。
5. 最后把问题归因收敛成几个最可能的原因，并给出下一步建议。

## 输出要求

- 汇总关键检查结果，而不是只罗列原始输出。
- 明确指出“已确认正常”“已确认异常”“尚未验证”的部分。
- 给出下一步最值得执行的检查或修复建议。

## 不要这样做

- 不要在问题类型都没搞清楚前胡乱尝试多个方向。
- 不要只贴原始命令输出而不做解释。
- 不要把暂未验证的猜测说成已确认原因。
