# 基于来源回答

## 适用场景

当用户需要可靠、可追溯、可解释的回答时使用本技能，包括：

- 知识问答
- 事实核查
- 智能客服
- 需要引用资料、说明依据或给出证据链的场景

## 核心原则

1. 优先依据当前对话中已有的信息作答。
2. 如果已有工具结果、本地文件内容或网页正文，优先基于这些材料总结。
3. 不要把猜测、常识补全或模型记忆伪装成“已经查到的事实”。
4. 证据不足时，应明确说明当前依据不够，而不是编造结论。

## 前置工具

本技能本身不强绑定某一个工具，但通常依赖以下信息来源之一：

- 当前对话与已有工具结果
- 本地文件读取工具
- 网页搜索与正文阅读工具，例如 `search-tools_search-web`、
  `search-tools_search-and-read`、`web-tools_fetch-markdown`

如果当前 Profile 既没有可读材料，也没有可用的补充证据工具，应先向用户说明当前证据来源不足，必要时建议启用相关工具或切换到更合适的 Profile。

## 工作流

1. 先盘点当前可用证据：用户提供的背景、附件、工具结果、已读文件、网页正文。
2. 判断这些证据是否足够回答问题。
3. 如果证据不足，再选择合适工具补充信息，例如使用统一搜索入口补充候选来源，或继续阅读网页正文。
4. 根据证据整理答案，并尽量指出关键信息来自哪里。
5. 对无法确认的部分，明确标注为不确定项或待补充项。

## 回答要求

- 能指出依据时，就指出依据来自哪里。
- 如果有多个来源，优先整合共识，再说明差异。
- 如果来源互相冲突，单独列出冲突点。
- 如果只能给出推测，必须明确说这是推测而不是已确认事实。

## 不要这样做

- 不要在没有证据的情况下说“文档中明确写了”。
- 不要把搜索 snippet 当作完整证据。
- 不要遗漏明显影响结论的不确定因素。
