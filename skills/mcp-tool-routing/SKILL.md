# MCP 与工具路由

## 适用场景

当你面对以下情况时，优先使用本技能：

- 当前可用工具较多，能力存在重叠
- 既有本地工具，也有 MCP 工具
- 需要判断该先调用哪类工具
- 某个工具失败后，需要选择替代路径

## 前置条件

本技能依赖当前 Profile 中已经配置了可选工具集合，至少应包含：

- 一组本地工具、MCP 工具或两者并存

如果当前 Profile 可用工具过少，无法形成有效路由选择，应直接说明“当前没有足够的候选工具可供路由”，而不是空谈路由策略。

## 选择原则

1. 先根据工具名称、描述和参数判断哪个工具最贴近当前任务。
2. 简单、直接、本地即可完成的任务，优先考虑本地工具。
3. 需要外部系统能力、专门协议或远程服务时，再考虑 MCP 工具。
4. 如果多个工具都能做同一件事，优先选输出更稳定、步骤更短、失败成本更低的那个。
5. 如果某个工具失败，不要机械重试同一调用；先判断是否应该换工具、换参数或换路径。

## 推荐流程

1. 明确当前任务需要的能力类型：读写文件、联网搜索、网络诊断、交互提问、外部服务等。
2. 在当前可用工具中选最贴近目标的 1 个主要工具。
3. 如有必要，再保留 1 个候选回退工具。
4. 执行后根据结果决定：
   - 成功：继续下一步
   - 失败：总结失败原因，再决定是否换工具或调整参数
5. 如果工具能力明显不足，应及时向用户说明缺口，而不是盲目尝试无关工具。

## 不要这样做

- 不要为了试错一次性调用很多相似工具。
- 不要把 MCP 工具当成默认更高级的选择。
- 不要在失败后无分析地重复同样的调用。
