# 规划并跟踪任务

## 适用场景

当任务满足以下特征时，优先使用本技能：

- 包含多个明显步骤
- 需要先调查再执行
- 中途容易遗漏状态或重复劳动
- 用户希望分阶段推进并确认结果

## 工作流

1. 先把任务拆成 3 到 10 个清晰步骤。
2. 如果任务足够复杂，调用 `todo-tools_init-todos` 初始化待办列表。
3. 每完成一个关键步骤，就调用 `todo-tools_update-todos` 更新状态。
4. 在进入下一阶段前，用 `todo-tools_read-todos` 检查是否有遗漏。
5. 最终总结时，说明哪些已完成，哪些仍待确认。

## 前置工具

优先工具：

- `todo-tools_init-todos`
- `todo-tools_update-todos`
- `todo-tools_read-todos`

如果当前 Profile 没有启用待办工具，本技能仍可退化为“先给简短计划，再在自然语言中同步进度”的方式，但不要伪造待办 ID 或假装已经调用了待办工具。

## 拆解原则

- 步骤要面向结果，不要写成过细的机械动作。
- 只把真正重要的节点放进待办，不要把每一个细小操作都写成待办。
- 如果任务很小，可以直接执行，不必为了形式强行建立待办。

## 输出建议

- 先给出简洁计划，再开始执行。
- 阶段完成后汇报实际结果，而不是只汇报“已完成步骤编号”。
- 如果任务被中断，要说明当前停在什么位置，以及下一步应该做什么。

## 不要这样做

- 不要为一个两步以内的小任务过度规划。
- 不要初始化待办后长期不更新。
- 不要把待办当成最终结果，真正重要的是执行产出与结论。
