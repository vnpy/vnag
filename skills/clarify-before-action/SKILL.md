# 先澄清再行动

## 适用场景

当以下情况出现时，优先使用本技能：

- 用户目标不够明确
- 缺少执行所需的关键参数
- 范围边界不清楚
- 存在多种合理方案，需要用户先选方向
- 继续执行可能导致误改、误查或无效工作

## 澄清原则

1. 只问真正影响执行的问题，不要为小事反复追问。
2. 优先一次问清关键歧义，避免把一个简单任务拆成很多轮。
3. 如果能从上下文合理推断，就直接说明假设并继续，而不是机械提问。
4. 如果工具或环境不支持交互式提问，应在回复中清楚列出需要用户补充的信息。

## 前置工具

优先工具：

- `interaction-tools_ask-user`

如果当前 Profile 没有启用交互提问工具，应改为在回复正文中清楚列出需要用户补充的信息，不要假装已经完成了澄清。

## 推荐流程

1. 先判断当前信息是否足够开始。
2. 如果不够，列出最关键的 1 到 3 个缺口。
3. 优先使用 `interaction-tools_ask-user` 提问：
   - 开放式问题适合收集背景
   - 选项式问题适合让用户在几种方案中选择
4. 拿到回答后，确认约束是否已经足够。
5. 再进入后续搜索、读取、执行或修改流程。

## 提问建议

- 问题要具体，避免“请再详细说说”这种泛问题。
- 如果有明确选项，优先给选项。
- 如果用户可能有其他需求，可允许补充自由输入。

## 不要这样做

- 不要在已有足够信息时仍继续追问。
- 不要一次提出太多无关问题。
- 不要在关键条件未明确时贸然执行高成本操作。
